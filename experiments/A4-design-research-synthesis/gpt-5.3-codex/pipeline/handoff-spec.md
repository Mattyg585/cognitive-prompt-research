# Pipeline Specification: A4 Design Research Synthesis

## Overview

This Tier 3 pipeline separates incompatible modes so synthesis can move beyond theme cataloguing into strategic reframing:

```
stage-1-data-reader -> stage-2-pattern-synthesiser -> stage-3-strategic-translator
```

## Agent map

| Stage | Function | Thinking modes | Input | Output |
|---|---|---|---|---|
| stage-1-data-reader | Capture evidence per participant | Investigation + light structuring | Raw interviews | Structured participant observations |
| stage-2-pattern-synthesiser | Build cross-participant pattern map | Synthesis + structuring | Stage 1 output only | Themes, evidence map, tensions, unresolved questions |
| stage-3-strategic-translator | Translate patterns into product strategy | Reframing + generation + light evaluation | Stage 2 output only | Final synthesis report |

## Handoff boundaries

### Handoff 1 (Stage 1 → Stage 2)

**Pass**
- Per-participant quotes
- Behavior notes
- Pain points, workarounds, emotions, contradictions

**Drop**
- Reader’s exploratory process narrative
- Early cross-participant interpretations

**Why**
- Stage 2 should pattern-match, not re-investigate.

### Handoff 2 (Stage 2 → Stage 3)

**Pass**
- Themes with evidence strength
- Cross-theme relationships
- Tensions and open questions

**Drop**
- Alternative clustering debates
- Synthesis-side deliberation

**Why**
- Stage 3 should reframe for decision-making, not reopen clustering.

## Execution rule (strict)

- Stage 2 may only read Stage 1 output.
- Stage 3 may only read Stage 2 output.
- No back-reading to raw interviews in downstream stages.

## Why this should beat monolithic prompting

A4’s failure mode is template-shaped catalogue output. This pipeline withholds evaluative and audience framing constraints until after pattern formation, increasing the chance of strategic reframing.
