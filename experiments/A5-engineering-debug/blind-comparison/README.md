# Blind Comparison: A5 — Engineering Debug

## Task

Debug an intermittent order processing failure in production. See `test-material.md` for the bug report with error messages, metrics, and system context.

## Evaluation Criteria

- **Depth**: Does the analysis go beyond the proximate cause? Does it identify systemic factors?
- **Specificity**: Are findings tied to specific evidence (error messages, metrics, code paths)?
- **Completeness**: Does it cover reproduction, isolation, diagnosis, and fix? Are edge cases considered?
- **Audience Awareness**: Would this be useful to the on-call engineer? To the team doing a post-incident review?
- **Correctness**: Does the analysis correctly identify the root cause and propose a valid fix?
