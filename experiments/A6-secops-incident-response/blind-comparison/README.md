# Blind Comparison: A6 — SecOps Incident Response

## Task

Write a blameless postmortem for a resolved SEV1 incident: customer data exposed via misconfigured API endpoint. 11-day exposure window, 2,847 records affected. See `test-material.md` for full incident data.

## Evaluation Criteria

- **Depth**: Does the postmortem go beyond describing what happened? Does it identify systemic vulnerabilities and organisational patterns?
- **Specificity**: Are findings tied to specific timeline events and evidence?
- **Completeness**: Does it cover timeline, root cause, response evaluation, action items, and lessons learned?
- **Audience Awareness**: Would this serve the engineering team, security team, and leadership? Does it enable organisational learning?
- **Vulnerability Detection**: Does it identify the genuinely systemic issues (not just the surface annotation error)?
