

# Cognitive Science Foundations for a Prompt Architect Agent

## The Core Claim Under Investigation

"Context carries cognitive mode" -- when you mix incompatible types of thinking (e.g., open investigation + evaluative judgment) in the same AI context window, the output degrades. This was discovered empirically by building AI pipelines. The question is whether established cognitive science supports this finding, and where the analogies hold vs. break down.

---

## 1. Cognitive Load Theory (Sweller)

### What the science actually says

Sweller's Cognitive Load Theory identifies three types of load on working memory:

- **Intrinsic load**: determined by the inherent complexity of the material and the number of interacting elements that must be processed simultaneously. A task with many elements that interact with each other (you can't process one without considering the others) has high intrinsic load. This is fixed by the nature of the task itself.

- **Extraneous load**: generated by suboptimal task presentation or design -- consumes working memory without contributing to the goal. Unclear instructions, irrelevant information, poorly organized materials all generate extraneous load. This is the load that instructional designers try to minimize.

- **Germane load**: the productive effort spent building schemas and organizing knowledge for long-term retention. This is the "good" load -- the mental work of actually learning and integrating information.

The critical insight is that these loads are **additive and compete for the same finite working memory resource**. Total cognitive load = intrinsic + extraneous + germane. When extraneous load is high, it directly reduces the capacity available for intrinsic processing and germane schema-building. You cannot increase one without decreasing the available budget for the others.

### Mapping to AI prompts -- strong

This maps well to "context carries cognitive mode." When a prompt mixes investigation and evaluation, the evaluation framing acts as extraneous load on the investigation task (and vice versa). The model has to "hold" both cognitive framings simultaneously, and they compete for attention/processing capacity.

A 2025 benchmarking paper ([arXiv:2509.19517](https://arxiv.org/pdf/2509.19517)) directly applied CLT to LLMs, proposing a tripartite model where "Context Saturation" (task-irrelevant information) and "Attentional Residue" (interference from task-switching) degrade performance. They found that when combined load exceeds a **"fragility tipping point,"** LLMs transition from successful reasoning to **catastrophic failure** -- not graceful degradation. The paper placed particular emphasis on the pivotal role that extraneous cognitive load plays in instigating these performance failures.

Another paper ([arXiv:2506.06843](https://arxiv.org/html/2506.06843v1)) on coordinating multiple LLM agents found that maintaining complex, predefined role personas imposed more extraneous cognitive load on LLM agents than flexible thinking styles. Adherence to a rigid persona consumed processing capacity that could otherwise go to the actual task.

A third paper proposed a zero-shot, prompt-based framework called CLAI-Prompt that operationalizes intrinsic, extraneous, and germane load into computable LLM metrics and dynamically adjusts inference strategy to maximize "cognitive economy." This requires no model retraining -- it works purely through meta-prompt design.

### Where the mapping breaks down

CLT was designed for human instructional design, where the goal is learning and schema-building. **LLMs don't learn during inference** -- there is no "germane load" equivalent in a single inference pass. The model isn't building schemas for long-term retention; it's producing output. The germane load concept doesn't translate cleanly.

The mapping is strongest for intrinsic vs. extraneous load competition. When you add evaluation framing to an investigation prompt, you are adding extraneous load -- processing burden that competes with the investigation task without contributing to it. That part of CLT transfers directly.

## 2. Dual Process Theory -- and Beyond Binary

### What the science actually says

Evans and Stanovich's (2013) canonical formulation in [Perspectives on Psychological Science](https://journals.sagepub.com/doi/10.1177/1745691612460685) defines:

- **Type 1 processes**: rapid, autonomous processes that yield default responses. Critically, this is **not a single system** but a heterogeneous set. Stanovich coined "TASS" -- The Autonomous Set of Systems -- to emphasize that Type 1 encompasses many different automatic processes (learned associations, innate modules, overlearned skills) that share the property of being fast and not requiring working memory, but are otherwise quite different from each other.

- **Type 2 processes**: distinctive higher-order reasoning that supports hypothetical thinking and loads heavily on working memory. The defining feature is the ability to decouple from immediate stimuli and reason about possibilities.

Even the originators acknowledged this is not strictly binary. Evans and Stanovich themselves state it is "implausible that the gamut of autonomous processes labelled as Type 1 could in any meaningful way be described as belonging to a single system."

### Extensions beyond binary

Stanovich (2009) proposed splitting System 2 into a **tripartite model** in a paper titled ["Distinguishing the reflective, algorithmic, and autonomous minds: Is it time for a tri-process theory?"](https://www.researchgate.net/publication/237337356_Distinguishing_the_reflective_algorithmic_and_autonomous_minds_Is_it_time_for_a_tri-process_theory):

- **Autonomous mind** (Type 1): rapid pattern matching, heuristics, learned automaticities
- **Algorithmic mind**: the computational efficiency of problem-solving (what IQ tests measure) -- the raw processing power available for Type 2 thinking
- **Reflective mind**: the disposition to actually engage the algorithmic mind -- metacognitive monitoring and rational thinking dispositions. This is not captured by IQ tests. Someone can have high algorithmic capacity but low reflective disposition (they could reason well but don't bother to).

Sternberg's "Mental Self-Government" theory goes much further, proposing multiple independent dimensions of thinking styles: three functions (executive, legislative, judicial), four forms (hierarchical, monarchic, oligarchic, anarchic), two scopes (internal, external), and two leanings (liberal, conservative). This yields 13+ distinct thinking style dimensions -- far richer than any binary model.

Varga and Hamburger (2014) proposed a "tri-dimensional way" varying abstractness, divergence, and context-specificity as independent dimensions rather than treating thinking modes as discrete categories.

### Mapping to 6-7 stances -- partially supported

The practitioner's discovery of 6-7 distinct stances (investigation, structuring, evaluation, synthesis, reframing, generation, orchestration) is not directly validated by dual process theory, but it is **consistent with the direction the field has moved**. The field has moved away from strict binary toward acknowledging multiple subsystems and modes. Stanovich's tripartite model and Sternberg's multi-dimensional taxonomy both suggest that "two types" was always a simplification.

The practitioner's empirical stances may be a task-specific decomposition that is defensible but not directly derived from existing theory. The strongest claim you can make is: "Cognitive science has moved beyond binary models of thinking, and multi-modal frameworks are well-established. The specific modes we identified are empirically grounded in CTA of AI pipeline work."

### Where the mapping breaks down

Dual process theory is about the **architecture of human cognition**. LLMs don't have a Type 1/Type 2 distinction in their processing -- there is no "fast automatic" vs. "slow deliberate" mechanism in a single forward pass. Every token prediction involves the same transformer computation regardless of whether the output "looks like" System 1 or System 2 thinking.

The mapping works better as an analogy for **prompt framing** (a prompt biases the model toward one mode of output) than as a literal cognitive architecture parallel. When you prompt for "quick intuitive reaction" vs. "step-by-step logical analysis," you get different outputs, but the underlying computation is the same kind of process.

## 3. Guilford's Convergent/Divergent Thinking (1956)

### What the original research actually said

Guilford's 1956 "Structure of Intellect" (SOI) model, published in Psychological Bulletin (53, 267-293), was **far more than a convergent/divergent binary**. He proposed a three-dimensional model crossing operations, contents, and products. The five mental operations were:

1. **Cognition** -- understanding, comprehending, discovering, recognizing information
2. **Memory** -- encoding and retaining information for later retrieval
3. **Divergent Production** -- generating multiple possible answers from available information; characterized by fluency (quantity of ideas), originality (unusualness), flexibility (variety of categories), and elaboration (detail)
4. **Convergent Production** -- searching for and producing the single correct or best answer through logical deduction and inference
5. **Evaluation** -- making judgments about quality, correctness, adequacy, or desirability; comparing information against criteria

These five operations crossed with content types (figural, symbolic, semantic, behavioral) and product types (units, classes, relations, systems, transformations, implications) to yield 120+ distinct intellectual abilities. The popular convergent/divergent binary that permeates design thinking and creativity literature is a dramatic oversimplification of Guilford's actual model.

### Mapping to the practitioner's usage

The practitioner's use of "convergent" (compress, verify, ground) and "divergent" (explore, synthesize, follow threads) **partially maps to Guilford but oversimplifies his model** in a specific way. Guilford's "convergent production" is specifically about finding the single right answer through logical deduction. The practitioner's "convergent" seems closer to Guilford's "evaluation" combined with convergent production -- compressing and verifying involves judgment (evaluation) as well as deduction (convergent production).

The practitioner's "divergent" maps well to Guilford's divergent production -- generating multiple possibilities, following threads, exploring.

More importantly, **Guilford already had five operations, not two**. The practitioner's instinct that there are more than two modes is actually closer to Guilford's original model than the popular convergent/divergent binary that the field extracted from it. Guilford would have recognized:
- Investigation as **cognition**
- Structuring as **convergent production**
- Evaluation as **evaluation**
- Synthesis/generation as **divergent production**
- Orchestration as a meta-operation not in his model (though it could map to aspects of evaluation applied to process rather than content)

A 2025 paper by Kramer ([arXiv:2503.22036](https://arxiv.org/html/2503.22036)) directly applies Guilford's full SOI model to LLM prompt engineering, using all five cognitive operations as structured prompt components. This improved GPT-4.1 performance on AIME2024 math problems from 26.7% to 43.3% -- a substantial gain from structuring prompts according to cognitive operation types rather than treating all reasoning as one mode.

### Where the mapping breaks down

Guilford's model is about **human intellectual abilities**, not about task framing or cognitive modes that interfere with each other. The SOI model is a taxonomy of what the mind can do, not a theory about what happens when you try to do multiple things at once. It doesn't predict that mixing operations degrades performance -- it classifies operations as distinct but doesn't make interference claims.

You need CLT or task-switching research for the interference claim. Guilford gives you the vocabulary for naming the modes; other theories give you the mechanism for why mixing them causes problems.

## 4. Double Diamond / Convergent-Divergent Cycling

### What the established framework says

The Double Diamond was developed by the British Design Council in 2005, adapted from Banathy's (1996) divergence-convergence model. It has four phases:

1. **Discover** (diverge) -- explore the problem space broadly, gather insights, understand user needs without filtering
2. **Define** (converge) -- synthesize findings into a clear, focused problem definition
3. **Develop** (diverge) -- generate many possible solutions, ideate broadly
4. **Deliver** (converge) -- refine, test, and implement the best solution

The critical design principle embedded in the Double Diamond: **"Trying to diverge and converge at the same time is like driving with the brakes on."** Each phase has a single cognitive mode, and transitions between phases are explicit.

The model has been extended by various practitioners. The "Triple Diamond" adds an initial discovery phase. The "Double Diamond with Iteration" allows looping back. But the core principle -- separate divergent and convergent phases -- remains constant across all variants.

### Mapping to the practitioner's pipeline

The practitioner's pattern (compress data -> reason over it -> compress findings -> reason over those) is **structurally similar to the Double Diamond but not identical**. The Double Diamond alternates between divergent exploration and convergent definition. The practitioner's pattern alternates between convergent compression and divergent reasoning -- **the phases are inverted** relative to the classic model.

The Double Diamond goes: diverge -> converge -> diverge -> converge.
The practitioner's pipeline goes: converge -> diverge -> converge -> diverge.

This inversion makes sense because the practitioner is working with **existing data that needs compression before reasoning**, whereas the Double Diamond assumes you start with an open problem space where you need to explore before you can define. The practitioner starts data-rich and needs to compress; the designer starts data-poor and needs to discover.

The pattern could be described as an **inverted Double Diamond** or a **compression-expansion pipeline**. The key shared insight -- that you must not mix divergent and convergent operations in the same phase -- is identical in both models.

### Where the mapping breaks down

The Double Diamond is a **design process model**, not a cognitive science finding. It is prescriptive (a methodology for organizing design work), not descriptive (an empirical finding about how cognition works). It was developed from observation of design practice, not from controlled experiments.

It provides a useful structural analogy for pipeline design but not scientific backing for why mode-mixing degrades output. The "why" comes from CLT and task-switching research. The Double Diamond gives you the "what" -- the pattern of alternating modes -- but not the mechanism.

## 5. Cognitive Task Analysis (CTA)

### What CTA actually is

CTA is a family of methods for eliciting the cognitive processes, knowledge structures, decision strategies, and mental models that experts use in complex tasks. It emerged from cognitive psychology and human factors engineering in the 1980s-90s. Key methods include:

- **Critical Decision Method (CDM)**: structured interviews using a specific incident as an anchor, probing for decision points, cues, and alternative assessments
- **Applied Cognitive Task Analysis (ACTA)**: a streamlined version with three components -- task diagram, knowledge audit, and simulation interview
- **GOMS (Goals, Operators, Methods, Selection rules)**: formal modeling of cognitive procedures
- **Think-aloud protocols**: having experts verbalize their thinking during task performance

CTA specifically aims to surface **tacit knowledge** that experts cannot easily articulate. Research shows experts routinely omit 40-70% of key steps, cues, and decision strategies when teaching without CTA -- not because they're withholding, but because expertise compresses knowledge into automatic routines that are no longer consciously accessible.

ACTA specifically examines four cognitive areas: (1) difficult judgments and decisions, (2) attentional demands, (3) critical cues and patterns, (4) problem-solving strategies.

CTA has three phases: (1) knowledge elicitation (interviewing/observing experts), (2) data analysis (identifying cognitive processes, decision points, knowledge structures), (3) knowledge representation (mapping knowledge in transferable formats -- decision tables, concept maps, cognitive demand tables).

### Mapping to what the practitioner did

The practitioner used CTA to identify where expert thinking "shifts gears" and used those gear-shift boundaries to define AI agent boundaries. This is a **legitimate and defensible** application of CTA. Traditional CTA identifies cognitive subtasks to inform training design, interface design, or expert system development. Using it to define AI pipeline stage boundaries is a natural extension -- you are asking "where does the expert change cognitive mode?" and using those transition points as architectural boundaries.

This is methodologically sound. CTA was designed to capture exactly this kind of information -- the cognitive structure of expert work, including mode transitions that experts themselves may not be able to articulate without structured elicitation.

### Has anyone else done this for AI system design?

Based on thorough searching, **not directly**. The closest work:

- The **Cognitive Design Patterns paper** (Wray, Kirk, and Laird, [arXiv:2505.07087](https://arxiv.org/html/2505.07087v2), 2025) catalogs recurring patterns from cognitive architectures (ACT-R, SOAR) and maps them to LLM agent systems. But it uses architectural analysis of existing systems, not CTA as a methodology for designing new ones.

- **Multi-agent LLM frameworks** assign different personas to agents (optimist, skeptic, formal proof assistant), which is a form of cognitive specialization. But this specialization is done by engineering intuition, not by systematic CTA of how experts actually approach the problem.

- The **CoALA framework** (Cognitive Architectures for Language Agents) proposes modular memory and action components inspired by cognitive science, but doesn't use CTA to determine the decomposition boundaries.

- **OpenAI's practical guide to building agents** (2025) recommends task decomposition but doesn't reference CTA or any systematic method for identifying decomposition boundaries.

The practitioner's approach -- **using CTA specifically to identify cognitive mode boundaries and using those boundaries to define agent/pipeline stage boundaries** -- appears to be a novel application. This is potentially a genuine contribution worth articulating explicitly.

### Where the mapping breaks down

CTA was designed to capture **expert human cognition** for transfer to novices or systems. When applied to AI pipeline design, there is an implicit assumption that "how an expert thinks about a problem" is also "how an AI pipeline should be structured." This is reasonable but not proven.

It is possible that optimal AI pipeline decomposition follows different boundaries than human cognitive mode shifts. An LLM might be able to handle certain mode combinations that humans cannot (or vice versa). The CTA-derived boundaries are a strong starting hypothesis, but they should be validated empirically against the AI system's actual performance, not assumed to be optimal because they reflect human cognition.

## 6. Working Memory, Chunking, and Context Windows

### What the science says

Miller's 1956 paper "The Magical Number Seven, Plus or Minus Two" proposed that working memory can hold approximately 7 chunks of information. Cowan (2001) revised this downward to approximately **4 chunks** when strategies like rehearsal and grouping are controlled for.

**Chunking** is the process of organizing individual items into meaningful groups. Expert chess players don't remember individual piece positions -- they recognize patterns (chunks) of pieces that form meaningful configurations. This effectively expands working memory capacity by compressing information into higher-level units.

Key findings on interference in working memory:
- **Proactive interference (PI)**: earlier information disrupts processing of newer information. The more similar the earlier material is to the current material, the stronger the interference.
- **Retroactive interference (RI)**: newer information disrupts recall of earlier information.
- **Feature overwriting**: when items in working memory share features, they can overwrite each other, leading to "blending" errors.
- These interference effects **increase when items are semantically similar** -- which is precisely what happens when multiple cognitive modes operate on the same topic within one context.

### Mapping to context windows -- surprisingly strong

A 2025 paper titled [*"Unable to Forget: Proactive Interference Reveals Working Memory Limits in LLMs Beyond Context Length"*](https://arxiv.org/abs/2506.08184) adapted proactive interference paradigms from cognitive science directly to LLMs. Key findings:
- LLM retrieval accuracy **declines log-linearly toward zero** as interference from semantically related earlier content accumulates
- This happens **well within the context window limit** -- the degradation is not about running out of tokens but about semantic interference
- Larger models show greater resistance to interference, paralleling individual differences in human working memory capacity
- Attempts to use natural language prompts for "strategic forgetting" yielded only marginal improvements -- you cannot prompt your way out of proactive interference

A separate paper on [*"Cognitive Workspace"*](https://arxiv.org/html/2508.13171v1) for LLMs draws explicitly on Baddeley's working memory model, Clark's extended mind thesis, and Hutchins' distributed cognition to propose active memory management for LLM agents. They found that naive "add-all" approaches to context lead to **"catastrophic interference"** -- the same term used in the neural network literature for when learning new information destroys old information.

The chunking parallel is also productive: RAG systems that chunk documents without respecting semantic boundaries show 15-30% accuracy degradation, directly paralleling how poorly-formed chunks impair human working memory. Effective chunking in RAG -- like effective chunking in human cognition -- requires that chunk boundaries align with meaningful semantic units.

### Where the mapping breaks down

Human working memory and LLM context windows operate via **completely different mechanisms**. Human working memory involves active maintenance through rehearsal loops, attentional refreshing, and is constrained by temporal decay (information fades if not actively maintained). LLM context windows are attention-based, operate in parallel across all positions, and are constrained by positional encoding schemes and attention head capacity.

The behavioral parallels are striking -- both show proactive interference, both show degradation with semantic similarity, both show capacity limits. But the underlying mechanisms are entirely different. This means insights from cognitive science may transfer at the behavioral level (predictions about when performance will degrade) but not always at the mechanistic level (explanations of why it degrades or interventions to fix it).

The "strategic forgetting" finding is particularly telling: in human cognition, directed forgetting works (you can intentionally suppress information). In LLMs, prompting for forgetting doesn't work well -- the information is still in the context and still exerts interference through the attention mechanism. Different mechanisms, similar symptoms, different interventions.

## 7. Set Shifting / Task Switching Costs

### What the science says

Task switching research (Rubinstein, Meyer, Evans, 2001; Monsell, 2003) demonstrates measurable performance costs when switching between cognitive tasks:

- **Switch costs**: response times are 200-1000ms slower and accuracy drops on the first trial after a task switch compared to a task repetition. These costs persist even when subjects know in advance which task is coming (though advance preparation reduces them).

- **Two distinct stages** contribute to switch costs: **goal shifting** (updating the contents of working memory with the new task's goals and rules) and **rule activation** (enabling the new task's stimulus-response mappings while disabling the old task's mappings). Both stages take measurable time and are error-prone.

- **Task-set inertia**: the prior task's configuration persists in memory and interferes proactively with the new task, **even beyond voluntary control**. You cannot simply decide to stop being influenced by the previous task. The old task set "leaks" into the new task.

- **N-2 repetition costs**: a counterintuitive finding -- returning to a recently abandoned task (A-B-A pattern) is **slower** than switching to a completely new task (C-B-A pattern). This occurs because the abandoned task was actively inhibited to enable the switch to B, and that inhibition persists, making it harder to reactivate A. This has direct implications for pipeline design where you alternate between modes.

- **Mixing costs**: even within blocks where task switches are possible (but don't always occur), performance on repeat trials is worse than in pure blocks where only one task is performed. The mere possibility of switching degrades performance.

- A 2024 paper found that switching between **cognitive strategies** (not just stimulus-response tasks) also incurs switch costs, even when the abstract goal remains the same. This is particularly relevant -- it means switching from "investigate" to "evaluate" carries a cost even if both are directed at the same problem.

Estimates suggest task switching can reduce human productivity by up to 40% in complex cognitive work.

### Mapping to AI prompts -- this may be the strongest parallel

The [Cognitive Load Limits paper](https://arxiv.org/pdf/2509.19517) explicitly models **"Attentional Residue"** in LLMs as analogous to task-switching costs. When a prompt asks a model to investigate and then evaluate within the same context, the investigative framing creates residue that interferes with clean evaluation, and the evaluative framing prematurely narrows the exploration.

**Task-set inertia** maps directly to what happens when an LLM is asked to shift from open exploration to critical evaluation mid-prompt. The exploratory framing persists in the context and biases the evaluation toward leniency (because exploration generated the ideas, the model is reluctant to reject them). Or the evaluative framing, if introduced early, prematurely narrows the exploration (the model self-censors exploratory moves because it knows evaluation is coming).

The **n-2 repetition cost** finding is interesting for pipeline design: if you structure a pipeline as investigation -> evaluation -> investigation, the second investigation phase may be impaired because the investigative stance was actively inhibited during the evaluation phase. This suggests that simple alternation between two modes may be worse than using fresh contexts for each phase.

The **mixing costs** finding is relevant to prompts that say something like "investigate this, but also keep in mind that we'll need to evaluate the findings." Even this anticipatory framing may degrade the investigation -- the mere awareness that evaluation will follow changes how the model investigates.

### Attentional residue as a distinct construct

The task-switching literature contains three related but distinct phenomena that are often conflated. Separating them clarifies which aspects of mode-mixing map cleanly to LLMs and which do not:

- **Switch costs** (Monsell, 2003): The temporal reconfiguration overhead when switching between tasks -- the measurable delay and accuracy drop on the first trial after a switch. This involves updating working memory contents with new goals and enabling new stimulus-response mappings while disabling old ones. **LLM mapping: weak.** There is no temporal switching in a single forward pass. The model processes the entire context in parallel, not sequentially.

- **Attentional residue** (Leroy, 2009; applied to LLMs in "Cognitive Load Limits" arXiv:2509.19517, 2025): The contextual framing from a prior task *persists and biases* subsequent processing, even after the nominal task switch. This is not about reconfiguration time -- it is about the prior frame continuing to exert influence on the current task. **LLM mapping: very strong.** Evaluation criteria present earlier in the context actively bias investigation toward pre-filtered pattern matching. The prior framing does not decay -- it remains in the context window and continues to shape attention patterns throughout.

- **Proactive interference** ("Unable to Forget", arXiv:2506.08184, 2025): Earlier information in context disrupts retrieval and processing of later updates. This is a fundamental working-memory limitation, not a task-switching artifact. Prompt engineering provides only marginal relief -- you cannot instruct the model to "forget" earlier context. **LLM mapping: very strong.** Demonstrated empirically with log-linear decay patterns that parallel human working memory interference.

The key insight is that attentional residue from evaluation criteria doesn't just add extraneous load -- it **switches the decision architecture** from recognition-primed (pattern discovery) to criterion-referenced (pattern matching against known standards). This is a qualitatively different degradation than simple overload. The model is not merely "distracted" by evaluation framing -- it is operating in a fundamentally different cognitive mode, one that pre-filters investigation toward confirming or denying predefined categories rather than discovering emergent patterns in the data.

### Where the mapping breaks down

Human task-switching costs involve **temporal reconfiguration** of attentional control settings -- it takes time to shift from one task set to another, and the old set decays gradually. LLMs process the entire context in parallel during a forward pass. There is no temporal sequence of "first process the investigation part, then switch to the evaluation part."

The "switching" in an LLM is better understood as **competing attentional patterns within the same forward pass**, not a temporal reconfiguration. The attention mechanism must simultaneously attend to investigation-framing tokens and evaluation-framing tokens, and these exert competing influences on the output distribution.

The behavioral effect (degraded output when modes are mixed) may be similar, but the mechanism is fundamentally different. In humans, switching costs arise from reconfiguration time and inhibition persistence. In LLMs, the analogous effect arises from conflicting attention patterns and distributional interference.

## 8. Metacognition

### What the science says

Nelson and Narens (1990/1992) established the foundational framework for metacognition, which operates across two levels:

- **Object level**: carries out basic cognitive work (perceiving, reasoning, deciding, producing output)
- **Meta level**: contains a dynamic model of the object level and exercises two functions:
  - **Monitoring** (upward flow): the meta level observes and models what the object level is doing
  - **Control** (downward flow): the meta level directs, adjusts, or redirects the object level's processing

Metacognitive knowledge falls into three categories:
- **Declarative**: knowing what -- knowledge about yourself as a thinker, about tasks, and about strategies ("I know that I reason poorly when tired")
- **Procedural**: knowing how -- knowledge of how to use cognitive strategies ("I know how to break a complex problem into sub-problems")
- **Conditional**: knowing when and why -- knowledge of when particular strategies are appropriate and why they work ("I know to use divergent thinking when the problem space is poorly defined")

Ackerman and Thompson (2017) characterize **meta-reasoning** specifically as "a cobbled-together skill comprising diverse self-management strategies acquired through individual and cultural learning" that approximate monitoring-and-control functions. A key finding: metacognitive judgments are often based on **heuristic cues** (fluency, feeling of effort, elapsed time) rather than direct access to actual cognitive performance. People use proxies to judge how well they're thinking, and these proxies are often wrong.

Flavell's (1979) original taxonomy distinguishes metacognitive knowledge (what you know about cognition) from metacognitive experiences (the conscious feelings that accompany cognitive activity -- the "aha" moment, the "tip of the tongue" state, the feeling of confusion).

### Mapping to the prompt architect agent -- clean and direct

The prompt architect agent is performing metacognition in the Nelson and Narens sense:

- It **monitors** object-level prompts (the prompts it analyzes) -- this is the upward monitoring flow
- It maintains a **model** of what cognitive modes those prompts invoke -- this is the meta-level representation
- It exercises **control** by recommending restructuring -- this is the downward control flow

The three types of metacognitive knowledge map to what the prompt architect needs:
- **Declarative**: knowledge about which cognitive modes exist and what characterizes them
- **Procedural**: knowledge about how to detect mode conflicts in a prompt
- **Conditional**: knowledge about when mode separation is necessary and when mixing is tolerable

A 2025 paper ([Meta-R1, arXiv:2508.17291](https://arxiv.org/pdf/2508.17291)) directly applies the object-level/meta-level framework to LLM reasoning, decomposing reasoning into object-level computation and meta-level monitoring/control. They found that explicitly separating these levels improved both reasoning performance and token efficiency. The meta-level component could identify when the object-level reasoning was going astray and redirect it -- exactly the monitoring-and-control loop that Nelson and Narens described.

### Where the mapping breaks down

Human metacognition is **often inaccurate**. This is one of the most robust findings in the field. People are poor judges of their own cognitive processes -- the "metacognitive illusion" problem. Dunning-Kruger is one manifestation, but there are many others: people misjudge their confidence, their comprehension, their learning rate.

An AI metacognitive agent might face analogous issues: it might misidentify which cognitive mode a prompt invokes, or incorrectly assess whether two modes interfere in a given case. The framework (object level / meta level) is sound, but the **reliability of the metacognitive judgments** is an open question that needs empirical validation.

Additionally, human metacognition involves **phenomenal experience** -- the "feeling of knowing," "tip of the tongue" states, the subjective sense of confusion or clarity. These feeling-states serve as heuristic cues for metacognitive monitoring. The AI version operates purely on structural and linguistic analysis, which is a different mechanism for monitoring even if the functional role (detecting when processing is going well or poorly) is analogous.

The conditional metacognitive knowledge -- knowing when mode separation matters and when it doesn't -- is likely the hardest part to get right. There are cases where mixing modes is acceptable or even productive (e.g., evaluative judgment integrated into generation can improve quality in some cases). The prompt architect needs to know the difference, and this requires nuanced conditional knowledge that may be hard to encode.

## 9. Klein's Recognition-Primed Decision Making (RPD)

### What the science says

Gary Klein's naturalistic decision making research (1986, 1999) overturned the classical model of expert decision making. Studying firefighters, military commanders, and other experts making decisions under time pressure and uncertainty, Klein found that experts in naturalistic settings **don't compare options** -- they recognise situations as instances of familiar patterns and mentally simulate the first action that comes to mind. If the simulation reveals a problem, they modify the action or consider the next most typical response. They almost never generate and compare multiple options.

The RPD model has three variations on a gradient of increasing cognitive demand:

- **Variation 1 (fully familiar situation)**: Immediate pattern recognition leads directly to action. The expert perceives the situation, recognises it as a familiar type, and knows what to do. No deliberation is needed -- the response is a practiced, automatic consequence of recognising the pattern.

- **Variation 2 (novel situation, known repertoire)**: The situation is unfamiliar or ambiguous, but the expert has relevant methods and frameworks in their repertoire. The expert must first **diagnose the unfamiliar situation** -- building a mental model of what's happening -- before applying known methods. This requires deliberate situation modeling that Variation 1 does not.

- **Variation 3 (novel situation, novel action needed)**: Both the situation and the required response are unfamiliar. The expert must **both model the situation AND construct new approaches** by adapting, combining, or inventing action sequences. This is the highest cognitive demand -- it requires genuine creative problem-solving rather than recognition or application of known methods.

### How it maps to LLMs -- arguably the best single predictive framework

The RPD model is arguably the best single framework for predicting **when pipeline separation helps and when it hurts**:

- **Variation 1 = LLM reasoning from training knowledge.** When the task matches patterns extensively represented in training data (professional questions, known legal frameworks, standard financial analyses), the LLM's pattern recognition fires and the answer is correct. Pipeline separation in this case **interrupts fluid expert performance** -- it adds overhead, forces unnecessary decomposition, and may actually degrade output by preventing the model from leveraging its integrated knowledge.

- **Variation 2 = LLM facing unfamiliar data but having relevant analytical methods in training.** The model has seen this *type* of analysis but not this specific data. Pipeline helps because it forces the "diagnose first" step -- an investigation stage builds a situation model of the novel data before an analysis stage applies known methods to that model. Without the pipeline, the model may skip the situation-modeling step and pattern-match directly from training, producing plausible but shallow analysis.

- **Variation 3 = LLM needing to construct entirely new analytical approaches for novel data.** The model has neither seen this data pattern nor this analytical approach in training. Pipeline helps even more, because each stage can focus its full processing capacity on one aspect of the novel problem -- understanding the data, constructing an approach, applying and evaluating it.

### The boundary condition

Pipeline separation helps specifically at the **Variation 2/3 boundary** -- when the situation is novel enough that pattern recognition alone is insufficient and deliberate situation modeling is required. Below that boundary (Variation 1), pipeline adds cost without benefit. Above it (Variation 3), pipeline provides its greatest value.

This predicts a non-obvious result: **the same prompt architecture that improves output on novel tasks may degrade output on familiar tasks.** There is no universally optimal level of decomposition -- the optimal architecture depends on where the task falls on the RPD gradient.

### Where the mapping breaks down

Klein's model describes **temporal dynamics** of expert cognition -- pattern recognition happens first, then situation assessment, then mental simulation. LLMs process in parallel within a forward pass. The "recognition → assessment → simulation" sequence in RPD is a temporal unfolding that doesn't have a direct architectural analog in transformer computation.

The mapping works best as a predictor of **when decomposition adds value** rather than as a description of how the LLM processes information. RPD tells you which tasks benefit from pipeline separation; it doesn't tell you how the LLM is "thinking" about those tasks.

### Key papers

- Klein, G. (1999). *Sources of Power: How People Make Decisions*. MIT Press.
- Klein, G., Calderwood, R., & Clinton-Cirocco, A. (1986). Rapid decision making on the fire ground. *Proceedings of the Human Factors Society*, 30, 576-580.

## 10. Kahneman-Klein Boundary Conditions for Valid Intuition

### What the science says

The Kahneman-Klein adversarial collaboration (2009) is one of the most important papers in judgment and decision making. Daniel Kahneman (skeptic of expert intuition, emphasizing biases and heuristics) and Gary Klein (champion of expert intuition, emphasizing naturalistic decision making) spent years trying to identify precisely where they disagreed. Their conclusion: they largely agreed on the **boundary conditions**.

Intuitive expertise (pattern recognition that produces correct judgments without deliberate analysis) is trustworthy when **two conditions are met**:

**(a) The environment has stable, learnable regularities.** The domain must have genuine patterns that connect observable cues to correct actions. Chess has this; predicting stock prices largely does not. The environment must provide valid cues that are reliably associated with outcomes.

**(b) The expert has had extensive opportunity to learn those regularities.** Prolonged practice with prompt, clear feedback is necessary. The expert must have encountered enough cases to internalize the patterns. Without sufficient learning opportunity, even a regular environment won't produce valid intuition.

When both conditions are met, expert pattern recognition is trustworthy. When either condition fails -- the environment is irregular (low-validity) or the expert lacks sufficient learning opportunity -- intuitive judgments are unreliable and deliberate analytical methods should be preferred.

### How it maps to LLMs -- directly predictive

For an LLM, **training data IS the learning opportunity**. The model's "intuitive expertise" -- its ability to pattern-match from training knowledge -- is valid precisely when:

(a) The task domain has stable regularities that are well-represented in training data (legal analysis, standard financial reasoning, established scientific frameworks), AND

(b) The model has encountered extensive examples of correct reasoning in that domain during training.

When both conditions are met, the model's "intuitive" pattern-matching is valid -- a monolithic prompt that lets the model leverage its training knowledge will perform well, and pipeline separation adds overhead without corresponding benefit.

When either condition fails -- the data is novel (an "environment" the model has NOT learned), or the domain is underrepresented in training -- then forced deliberation through pipeline separation helps by replacing unreliable pattern recognition with structured analysis.

### The prediction

This directly predicts a clean experimental outcome:

- **Recognition-primed tasks** (matching training regularities) → Tier 2 (optimised prompt) is sufficient. The model's pattern recognition is valid for these tasks, and optimising the prompt to reduce extraneous load is all that's needed.

- **Investigation-required tasks** (novel data, underrepresented domains) → Tier 3 (pipeline reconstruction) earns its cost. The model's pattern recognition is unreliable for these tasks, and forced deliberation through separate investigation and analysis stages produces qualitatively better output.

### Where the mapping breaks down

Kahneman and Klein were discussing **individual expertise developed through personal experience**. An LLM's "expertise" comes from statistical patterns across billions of documents -- a fundamentally different kind of learning. The model has never "practiced" in the way Klein describes -- it has never made a decision, observed the outcome, and adjusted its approach.

The boundary conditions may still predict when pattern-matching works, but the *mechanism* of pattern acquisition is entirely different. Human experts develop intuition through repeated cycles of action and feedback; LLMs develop statistical associations through gradient descent on text. The behavioral predictions may transfer even though the learning mechanism does not.

### Key paper

- Kahneman, D. & Klein, G. (2009). Conditions for intuitive expertise: A failure to disagree. *American Psychologist*, 64(6), 515-526.

## 11. Bereiter & Scardamalia -- Knowledge-Telling vs Knowledge-Transformation

### What the science says

Bereiter and Scardamalia's (1993) research on expert writing revealed a counterintuitive finding: **expert writers work harder than novices, not easier.** Their model distinguishes two fundamentally different composing processes:

- **Knowledge-telling**: The novice strategy. The writer retrieves relevant knowledge from memory and writes it down in roughly the order it's retrieved. The process is: get a topic cue → retrieve associated content → write it → use what was just written as the next retrieval cue → repeat. This is cognitively economical -- it requires minimal planning, restructuring, or self-monitoring. The text flows directly from memory to page with little transformation.

- **Knowledge-transformation**: The expert strategy. The writer doesn't simply retrieve and record -- they actively **restructure the problem space and reconstruct their understanding** through the act of writing. The writer maintains two problem spaces simultaneously: a content space (what they know about the topic) and a rhetorical space (how to communicate effectively to the audience). The interaction between these two spaces drives genuine transformation of the writer's understanding -- writing becomes a tool for thinking, not just a record of thinking already done.

The key finding: expertise in writing doesn't just produce *better* text by applying better techniques to the same process. It produces better text by engaging in a **qualitatively different cognitive process** -- one that is more effortful, more recursive, and more genuinely transformative of the writer's own understanding.

### How it maps to LLMs -- the monolithic prompt problem

Monolithic prompts let the LLM **knowledge-tell** -- dump training knowledge in retrieval order. The model receives a prompt, retrieves associated patterns from its training distribution, and generates text in roughly the order those patterns activate. This is the LLM equivalent of the novice writer's strategy: content flows from training to output with minimal restructuring.

Pipeline separation forces **knowledge-transformation** -- each stage must reconstruct understanding from the previous stage's output rather than pattern-matching from training. When an investigation stage produces findings and a separate analysis stage must work with those findings (not with its own training priors), the analysis stage is forced into the rhetorical-content interaction that Bereiter and Scardamalia identified as the hallmark of expert processing. It must genuinely engage with the specific data rather than defaulting to generic training patterns.

### The boundary condition

This framework predicts when pipeline separation is **wasteful versus valuable**:

- **When knowledge-telling is appropriate** -- applying known frameworks to familiar problems, answering questions with established answers, producing standard professional documents -- pipeline separation adds cognitive overhead without corresponding benefit. The LLM's retrieval-order output is correct because the task doesn't require transformation.

- **When genuine knowledge transformation is needed** -- analyzing novel data that doesn't match training patterns, producing insights that require restructuring understanding, synthesizing across disparate domains -- pipeline separation forces the transformative process that a monolithic prompt allows the model to skip.

### Where the mapping breaks down

Bereiter and Scardamalia's model is about **learning through writing** -- the knowledge-transformation process changes the writer's understanding, and this changed understanding persists. LLMs don't learn during inference. The "transformation" in a pipeline is transformation of the output, not of the model's understanding. Each stage produces a transformed artifact, but the model itself is unchanged.

The analogy is strongest for the **process structure** (retrieval-based vs. reconstructive processing) rather than for the **outcome** (learning). Pipeline separation produces output that looks like it went through knowledge-transformation, and that output may be qualitatively better, but the model hasn't actually "learned" anything in the process.

### Key paper

- Bereiter, C. & Scardamalia, M. (1993). *Surpassing Ourselves: An Inquiry into the Nature and Implications of Expertise*. Open Court Publishing.

## 12. Epistemic Cognition (Chinn, Buckland & Samarapungavan)

### What the science says

Chinn, Buckland, and Samarapungavan (2011) proposed a multi-dimensional framework for epistemic cognition that goes well beyond earlier models (like Perry's developmental scheme or Kuhn's epistemological levels). Their framework identifies five interacting components:

- **Epistemic aims and values**: What the thinker is trying to achieve epistemically -- truth, understanding, explanation, certainty, simplicity, or other goals. Different aims drive fundamentally different cognitive processes.

- **Structure of knowledge**: How the thinker represents knowledge -- as isolated facts, as interconnected webs, as models, as theories. The perceived structure shapes how new information is integrated.

- **Sources and justification**: What counts as legitimate evidence and valid reasoning. Some thinkers rely on authority, others on empirical observation, others on logical coherence. The epistemic standards applied determine what gets accepted and what gets rejected.

- **Epistemic virtues and vices**: Intellectual character traits that support or undermine good epistemic practice. Virtues include intellectual humility, thoroughness, open-mindedness, and willingness to revise. Vices include dogmatism, premature certainty, confirmation bias, and intellectual laziness.

- **Reliable and unreliable processes**: The actual cognitive processes used -- some are reliably truth-conducive (careful observation, logical reasoning, systematic testing), others are unreliable (wishful thinking, reliance on anecdote, pattern-matching without verification).

The discourse-theoretic view, developed by Knight and Littleton and integrated into this framework, argues that epistemic cognition is not purely internal -- **it emerges through language and discourse patterns**. The way people talk about knowledge shapes how they think about knowledge. Exploratory talk (tentative, questioning, considering alternatives) produces different epistemic cognition than disputational talk (assertive, defending positions, seeking to win).

### How it maps to LLMs -- the epistemic stance intervention

The "epistemic stance" intervention in the cognitive stack -- setting "explore before concluding" in the system prompt or at the beginning of a pipeline stage -- is literally an intervention at the epistemic cognition level. It operates across multiple dimensions of Chinn et al.'s framework simultaneously:

- **Epistemic aims**: It sets the aim as exploration and understanding rather than closure and certainty. This changes what the model is "trying to do" at a fundamental level.

- **Epistemic virtues/vices**: It activates thoroughness, intellectual humility, and willingness to consider alternatives (virtues) while suppressing premature certainty, confirmation bias, and pattern-matching without verification (vices).

- **Reliable processes**: It biases toward careful observation and systematic analysis rather than rapid pattern-matching from training priors.

The discourse-theoretic view is particularly relevant here: **the language of the prompt shapes the epistemic stance of the output, because stance emerges through language patterns, not despite them.** When a prompt uses tentative, exploratory language ("let's investigate," "what patterns emerge," "consider multiple interpretations"), it activates the language patterns associated with exploratory epistemic cognition. When it uses assertive, evaluative language ("assess the quality," "determine whether," "rate on a scale"), it activates the patterns associated with closure-seeking epistemic cognition.

This is the mechanism by which "context carries cognitive mode" -- the epistemic stance encoded in the prompt's language patterns propagates through the model's output because the model learned the statistical associations between epistemic language patterns and the types of thinking they accompany.

### Where the mapping breaks down

Chinn et al.'s framework is about **genuine epistemic cognition** -- how people actually reason about knowledge, truth, and justification. LLMs produce text that *looks like* epistemic cognition but may not involve anything analogous to genuine epistemic processing. The model produces exploratory-sounding text not because it is genuinely exploring, but because the prompt activated language patterns associated with exploration.

Whether this distinction matters for output quality is an empirical question. If the output produced under exploratory epistemic framing is systematically better than output produced under closure-seeking framing (which the experiments aim to test), then the mechanism may not matter -- the intervention works even if the model isn't "really" engaging in epistemic cognition.

### Key paper

- Chinn, C. A., Buckland, L. A., & Samarapungavan, A. (2011). Expanding the dimensions of epistemic cognition. *Educational Psychologist*, 46(3), 141-167.

## 13. Spiro's Cognitive Flexibility Theory

### What the science says

Spiro, Feltovich, Jacobson, and Coulson (1992) developed Cognitive Flexibility Theory (CFT) to address a specific learning problem: how people acquire and apply knowledge in **ill-structured domains** -- domains where concepts interact differently across cases, where context radically changes how principles apply, and where simple schemas don't transfer cleanly from one case to another.

The core argument: learning in ill-structured domains requires **multiple representations** of the same material from different perspectives. Spiro calls this **"criss-crossing the conceptual landscape"** -- revisiting the same case or concept from different angles, highlighting different features, drawing different connections. This builds the flexible knowledge structures needed to handle novel cases where familiar concepts interact in unfamiliar ways.

The primary failure mode CFT identifies is **reductive bias**: the tendency to oversimplify complex, irregular knowledge into rigid schemas that work for textbook cases but fail for novel ones. Reductive bias manifests as:

- Treating complex cases as more similar to familiar cases than they actually are
- Applying a single schema when multiple interacting schemas are needed
- Ignoring contextual factors that change how principles apply
- Locking onto the first interpretation retrieved rather than considering alternatives

CFT distinguishes between two types of domains:

- **Well-structured domains**: Rules and principles transfer consistently across cases. A schema learned from one case applies cleanly to new cases. Multiple representations add little value because the single representation transfers reliably.

- **Ill-structured domains**: The same concept manifests differently across cases depending on context. Schema transfer is unreliable because contextual factors change how principles interact. Multiple representations are essential because no single representation captures the full range of concept-to-case mappings.

### How it maps to LLMs -- fighting reductive bias

Setting "explore before concluding" in a prompt or pipeline stage fights reductive bias directly -- it **prevents the model from locking onto the first schema retrieved** from training and instead forces consideration of multiple interpretations, alternative framings, and contextual factors that might change the analysis.

Pipeline separation forces the LLM to **criss-cross the same data from different cognitive angles**: an investigation stage examines the data for emergent patterns, an analysis stage applies frameworks, a synthesis stage integrates findings across perspectives. Each stage provides a different "traversal" of the conceptual landscape represented in the data, building the kind of flexible, multi-perspectival understanding that CFT identifies as necessary for ill-structured domains.

CFT predicts this matters most in **ill-structured domains with irregular concept-to-case mappings** -- precisely the domains where novel data is most likely to deviate from training patterns. Legal analysis of novel fact patterns, strategic analysis of unprecedented situations, scientific analysis of anomalous data -- these are ill-structured problems where reductive bias is most likely and where multiple perspectives add the most value.

For **well-structured domains with regular rules** -- standard contract review against established legal frameworks, financial analysis using known formulas, technical documentation following templates -- multiple perspectives add little because the schema transfers cleanly. The single representation (the known framework) is sufficient.

### Where the mapping breaks down

CFT is fundamentally a **learning theory** -- it's about how knowledge structures are acquired and organized for future flexible application. LLMs don't acquire new knowledge structures during inference. The pipeline doesn't build "flexible knowledge" in the model -- it produces output that reflects multiple perspectives.

The mapping works best as a description of **output quality**: pipeline-produced output resembles the product of cognitively flexible reasoning (multiple perspectives, contextual sensitivity, resistance to reductive bias), even though the model hasn't actually developed cognitive flexibility. Whether this difference matters depends on whether we care about the process or only the product.

### Key paper

- Spiro, R. J., Feltovich, P. J., Jacobson, M. J., & Coulson, R. L. (1992). Cognitive flexibility, constructivism, and hypertext: Random access instruction for advanced knowledge acquisition in ill-structured domains. In Duffy, T. M. & Jonassen, D. H. (Eds.), *Constructivism and the technology of instruction: A conversation* (pp. 57-76). Lawrence Erlbaum Associates.

## 14. Recent LLM + Cognitive Science Papers (2024-2026)

A growing body of work directly bridges cognitive science frameworks and LLM behavior, moving beyond analogy to empirical testing. Key papers:

### Dual-process theory applied to LLMs

A 2025 paper in *Nature Reviews Psychology* ("Dual-process theory and decision-making in large language models") systematically maps the System 1/System 2 distinction to LLM prompting strategies. The paper argues that standard prompting activates System 1-like pattern matching from training, while structured prompting techniques (chain-of-thought, step-by-step reasoning) activate System 2-like deliberative processing. Importantly, the paper identifies boundary conditions for when each approach is superior -- rapid pattern matching outperforms deliberation for well-practiced tasks, while deliberation outperforms pattern matching for novel or complex reasoning. This aligns directly with the RPD framework (Section 9) and the recognition-primed vs investigation-required distinction.

### Cognitive Load Limits in LLMs

The "Cognitive Load Limits in LLMs" paper (arXiv:2509.19517, 2025) formalises two computational analogs of extraneous cognitive load:

- **Context Saturation**: task-irrelevant information in the context window that consumes processing capacity without contributing to the target task. This is the LLM analog of extraneous cognitive load in Sweller's framework.

- **Attentional Residue**: interference from task-switching within a single context -- the framing from a prior task persists and biases subsequent processing. This is the LLM analog of task-set inertia from the switching literature.

The paper demonstrates that when combined load exceeds a "fragility tipping point," LLMs transition from successful reasoning to catastrophic failure -- not graceful degradation. This non-linear failure mode is particularly insidious because it means performance looks fine up until the point where it collapses entirely.

### Proactive Interference in LLMs

"Unable to Forget: Proactive Interference Reveals Working Memory Limits in LLMs Beyond Context Length" (arXiv:2506.08184, 2025) adapted proactive interference paradigms from cognitive psychology directly to LLMs. Key findings: LLMs exhibit proactive interference patterns that closely parallel human working memory limits. Earlier information in context disrupts retrieval of later updates, with log-linear decay. Critically, prompt-based interventions for "strategic forgetting" provided only marginal relief -- this is a fundamental architectural limitation, not a prompting problem.

### Multi-agent coordination under CLT

"United Minds or Isolated Agents?" (arXiv:2506.06843, 2025) applies Cognitive Load Theory directly to multi-agent LLM coordination. The paper tests whether cognitive load principles predict when multi-agent separation helps versus hurts. Key finding: maintaining complex, predefined role personas imposed more extraneous cognitive load than flexible thinking styles. Rigid persona adherence consumed processing capacity that could otherwise go to the actual task. This supports the design principle that pipeline stages should be defined by cognitive function (what type of thinking is needed) rather than by persona (who is doing the thinking).

### Implications for this research

These papers collectively demonstrate that cognitive science frameworks are not merely useful analogies for understanding LLM behavior -- they are **empirically predictive**. The CLT tripartite model, proactive interference from working memory research, and the System 1/System 2 distinction all generate testable predictions about LLM performance that have been confirmed experimentally. This strengthens the theoretical foundation for the mode-separation hypothesis: if cognitive load, attentional residue, and proactive interference all degrade LLM output in ways that parallel their effects on human cognition, then interventions derived from cognitive science (separating incompatible cognitive modes into distinct contexts) should produce corresponding improvements.

## Overall Synthesis: What Holds Up and What's a Stretch

### Strongest mappings (empirically supported in AI/LLM research)

| Concept | Strength | Key Evidence |
|---------|----------|-------------|
| Working memory interference / context windows | Very strong | PI-LLM paper directly demonstrates proactive interference in LLMs with log-linear decay, well within context length limits |
| Cognitive load (intrinsic vs extraneous) | Strong | Multiple 2024-2025 papers apply CLT to LLMs; "fragility tipping point" with catastrophic (not graceful) failure empirically demonstrated |
| Task switching / attentional residue | Strong | Directly modeled in LLM cognitive load benchmarks; behavioral parallels confirmed; n-2 repetition costs have pipeline design implications |
| Metacognition (object/meta levels) | Strong | Meta-R1 applies this directly to LLM reasoning with measurable improvements in both accuracy and efficiency |
| Guilford's multi-operation model | Moderate-strong | Kramer (2025) applies full SOI to prompt engineering with 16.6 percentage point performance gain |

### Reasonable analogies (structurally sound but mechanistically different)

| Concept | Strength | Notes |
|---------|----------|-------|
| CTA for pipeline decomposition | Moderate-novel | Legitimate methodology extension; appears to be a novel application; needs empirical validation that human cognitive mode boundaries are optimal for AI pipeline boundaries |
| Double Diamond cycling | Moderate | Good structural analogy for pipeline design; practitioner's pattern is an inverted Double Diamond; shared principle that modes must not be mixed within phases |
| Stanovich's tripartite mind / Sternberg's multi-dimensional styles | Moderate | Supports "more than two modes" argument; provides theoretical legitimacy for multi-stance frameworks; doesn't directly map to LLM architecture |

### Stretches (use with caveats)

| Concept | Issue |
|---------|-------|
| Dual Process Theory as literal LLM architecture | LLMs don't have System 1/System 2 processing; it's an output framing analogy only, not an architectural parallel |
| Germane cognitive load in LLMs | No learning occurs during inference; this component of CLT doesn't transfer |
| Chunking as literal working memory expansion in LLMs | The behavioral parallels exist (semantic boundaries matter for RAG), but the mechanism is engineering, not cognition |
| Phenomenal metacognitive experience | AI metacognition operates on structure, not feeling-states; the monitoring mechanism is fundamentally different even if the function is similar |

### The convergence argument

The practitioner's core insight -- **"context carries cognitive mode"** -- is best supported not by any single theory but by the **convergence of three established findings**:

1. **Extraneous cognitive load** (Sweller): mixing modes adds processing burden that competes with the actual task, and the competition is for a shared finite resource
2. **Task-switching costs** (Rubinstein, Monsell): shifting between cognitive stances incurs measurable performance degradation, prior stances persist as interference (task-set inertia), and even the anticipation of switching degrades current performance (mixing costs)
3. **Proactive interference in working memory** (demonstrated in LLMs by the PI-LLM paper): earlier contextual framing actively degrades processing of subsequent material, with log-linear decay, and this cannot be overcome by prompting for "forgetting"

Each theory alone would be an analogy. Together, they converge on the same prediction from different angles: **mixing cognitive modes in the same processing context will degrade output, and the degradation is not a matter of running out of space but of active interference between incompatible framings**.

The practitioner's discovery of 6-7 distinct stances is not directly derived from any single cognitive science framework, but it is most consistent with Guilford's five operations (1956) -- which is ironic, since it's the oldest framework in this list and the one whose richness has been most obscured by popular oversimplification. The novel contribution is using CTA to identify these stances empirically from expert AI pipeline work and then using the interference research to justify why they must be separated into distinct contexts.
